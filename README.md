# OpenCart Selenium E2E Automation Framework

## Tech Stack
- Java
- Maven
- Selenium WebDriver
- TestNG
- Rest Assured
- Git

## Project Overview
This is an End-to-End (E2E) automation framework built for the OpenCart application.
It validates complete user workflows using UI automation and backend verification.

## Key E2E Scenario Covered
- User Registration through UI (OpenCart Registration Form)
- Backend validation of registered user using API / database checks
- End-to-End verification from frontend to backend

## Framework Design
- Page Object Model (POM)
- Reusable utilities for browser setup and waits
- TestNG-based test execution
- Maven dependency management
- Backend validation using Rest Assured

## Example Flow
1. Launch OpenCart application
2. Fill and submit registration form via UI
3. Validate successful registration message
4. Verify user details through backend validation

## Purpose
This framework is created to practice real-world End-to-End automation testing
by combining Selenium UI automation with backend validation.
